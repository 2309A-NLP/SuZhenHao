# 基于 RAG 的角色扮演系统需求规格说明书（律师角色）

## 1. 项目概述
- 项目名称：角色扮演系统（法律专家版）
- 目标：构建一个可进行多轮对话的律师角色聊天机器人，基于法律条文与司法解释进行可追溯问答。
- 关键能力：RAG 检索增强、角色提示词模板、多用户多角色、短期与长期记忆、知识库动态更新。

## 2. 业务目标
- 为用户提供法律问题初步咨询，回答中给出法律依据与免责声明。
- 为教学/实验场景提供 RAG 全链路实践（检索、生成、评测、压测）。
- 支持后续扩展到更多领域角色（如财务顾问、医疗助手）。

## 3. 角色定义
- 终端用户：发起法律咨询，查看回复与引用来源。
- 系统管理员：维护知识库、配置模型、监控服务状态。
- 业务角色（AI）：律师（当前默认角色“张律师”）。

## 4. 功能需求
### 4.1 对话与角色扮演
- FR-01：系统支持用户输入问题并返回律师风格回答。
- FR-02：回答遵循提示词模板，包含法律依据、分析、建议、免责声明。
- FR-03：支持多轮上下文对话。

### 4.2 RAG 检索
- FR-04：对接 Milvus 知识库，基于法律条文/司法解释检索相关文本。
- FR-05：向量化模型使用 BGE-m3。
- FR-06：支持重排序模型 BGE-rerank（可配置）。
- FR-07：支持混合检索（稠密向量 + BM25 稀疏检索）。

### 4.3 记忆管理
- FR-08：短期记忆使用 Redis，保存最近会话（多会话隔离）。
- FR-09：长期记忆使用 Milvus，支持跨会话回忆用户信息。
- FR-10：短期记忆过期策略可配置（TTL）。

### 4.4 多用户多角色
- FR-11：支持多用户隔离会话数据。
- FR-12：支持同一用户多会话（session_id）。
- FR-13：支持多角色扩展（角色信息结构化存储）。

### 4.5 数据与存储
- FR-14：MySQL 存储用户信息与角色信息。
- FR-15：知识库支持增量更新，新增文档可向量化入库。

### 4.6 接口与测试
- FR-16：提供 HTTP JSON API（请求/响应格式明确）。
- FR-17：提供 Postman 测试集合。
- FR-18：支持 JMeter 压测并输出 QPS 指标。
- FR-19：支持 RAGAS 评测（答案相关性、上下文精确率等）。

## 5. 非功能需求
- NFR-01：可观测性：健康检查接口与错误日志。
- NFR-02：安全性：API Key 不得硬编码，使用环境变量。
- NFR-03：可扩展性：本地部署与在线 API 可切换（vLLM/SGLang/xInference/API）。
- NFR-04：性能：单实例 QPS 指标可压测，响应时延可统计。

## 6. 验收标准（简版）
- 用户可创建会话、切换会话并进行连续对话。
- 回答至少返回一条可追溯检索来源（命中时）。
- 系统在 Redis 不可用时可降级运行（短期记忆降级）。
- 文档完整：需求规格、设计文档、接口文档、评测与测试方案。
