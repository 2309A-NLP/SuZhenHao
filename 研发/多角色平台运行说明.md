# 多角色对话平台运行说明

## 1. 已修改的主要文件
- `main.py`：新增 `role_code`、`/roles`、`/knowledge/build`
- `chat_service.py`：改为 `MultiRoleChatService`
- `role_config.py`：9 个固定角色统一配置
- `prompts.py`：每个角色独立提示词模板
- `rag_retriever.py`：按角色路由 Milvus collection
- `memory.py`：短期/长期记忆加入 `role_code`
- `mysql_store.py`：会话与消息加入 `role_code`
- `mysql_schema.sql`：会话和消息表结构升级
- `build_role_kb.py`：自动抓取网页并写入角色知识库
- `frontend/index.html`
- `frontend/style.css`
- `frontend/app.js`

## 2. 安装依赖
在项目根目录执行：

```bash
pip install -r requirements.txt
```

## 3. MySQL 初始化
先创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS roleplay DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_general_ci;
```

再导入表结构：

```bash
mysql -uroot -proot roleplay < mysql_schema.sql
```

如果你的 `chat_sessions`、`chat_messages` 已经存在旧表，建议先手动补字段：

```sql
ALTER TABLE chat_sessions ADD COLUMN role_code VARCHAR(64) NOT NULL DEFAULT 'psychologist';
ALTER TABLE chat_sessions ADD COLUMN title VARCHAR(255) DEFAULT '新对话';
ALTER TABLE chat_sessions ADD COLUMN last_message TEXT;

ALTER TABLE chat_messages ADD COLUMN role_code VARCHAR(64) NOT NULL DEFAULT 'psychologist';
```

```bash

```

浏览器访问：

```text

```

## 6. 每个角色如何配置知识库
角色统一在 `role_config.py` 配置，字段如下：
- `role_code`
- `display_name`
- `subtitle`
- `icon`
- `icon_color`
- `greeting`
- `milvus_collection`

每个角色一个 Milvus collection，例如：
- `kb_doctor`
- `kb_lawyer`
- `kb_psychologist`
- `kb_teacher`
- `kb_scientist`
- `kb_stock_analyst`
- `kb_english_tutor`
- `kb_virtual_friend`
- `kb_financial_advisor`

## 7. 如何一键构建角色知识库
后端接口：

```http
POST /knowledge/build
Content-Type: application/json

{
  "role_code": "lawyer",
  "urls": [
    "https://www.example.com/page1",
    "https://www.example.com/page2"
  ]
}
```

或直接运行脚本：

```bash
python build_role_kb.py
```

如果你想改成自己的 URL 列表，直接修改脚本底部示例或在接口里传入。

## 8. 如何新增角色
1. 在 `role_config.py` 的 `ROLE_CONFIGS` 里新增角色元信息
2. 在 `ROLE_PROMPTS` 里新增对应提示词
3. 为新角色指定新的 `milvus_collection`
4. 准备该角色的知识来源 URL
5. 调用 `/knowledge/build` 或运行脚本导入知识
6. 前端会自动从 `/roles` 读取，无需再手写角色列表

## 9. 当前页面效果
- 登录前：左侧登录卡片 + 右侧欢迎页
- 登录后：三栏布局
  - 左侧：角色列表
  - 中间：当前角色对话窗口
  - 右侧：当前角色历史会话 + 新建对话

## 10. 注意事项
- 当前登录/注册仍是前端 `localStorage` 方式，不是后端鉴权
- 网页爬取会受反爬限制，建议优先用公开可访问、静态内容较多的网址
- 如果某个角色还没导入知识库，系统仍可回答，但主要依赖模型本身能力
