# 测试与评测方案（RAGAS + Postman + JMeter）

## 1. 功能测试（Postman）
- 目标：验证接口可用性与业务正确性。
- 用例：
  - 单轮问答：返回 `response` 和 `sources`
  - 多轮问答：后续问题能引用上下文
  - 多会话：`session_id` 隔离
  - 多用户：`user_id` 隔离
  - 清空会话：历史为空
  - 健康检查：`/health` 返回 `ok`

## 2. 压力测试（JMeter）
- 指标：
  - QPS（吞吐量）
  - P95/P99 响应时间
  - 错误率
- 场景建议：
  - 并发 20/50/100 用户逐级压测
  - 持续 5~10 分钟稳定性测试
  - 分别测试“仅检索”“检索+生成”两种模式

## 3. RAG 评测（RAGAS）
- 评测数据：法律问答样本（问题、参考答案、参考上下文）
- 指标：
  - faithfulness（忠实度）
  - answer_relevancy（答案相关性）
  - context_precision（上下文精确率）
  - context_recall（上下文召回率）
- 评测步骤：
  1. 构造评测集（JSON/CSV）
  2. 调用系统批量生成答案与检索上下文
  3. 使用 RAGAS 计算指标并输出报告

## 4. RAG 优化方向
- 检索层：
  - 稠密检索 + BM25 混合权重调优
  - 提升召回候选数后再重排序
- 重排序层：
  - 启用 BGE-rerank，观察 context_precision 提升
- 生成层：
  - 优化提示词模板结构（依据、分析、建议、免责声明）
  - 低温度提升事实一致性
- 数据层：
  - 知识库按法规时效版本管理
  - 增量更新后触发重建索引策略

## 5. 验收输出物
- Postman Collection（接口自动化测试）
- JMeter 脚本与压测报告（含 QPS）
- RAGAS 评测报告（含四项核心指标）
- 问题清单与优化迭代记录
