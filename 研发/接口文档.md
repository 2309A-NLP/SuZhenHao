# 接口文档（HTTP + JSON）

## 1. 基础信息
- Base URL：`http://localhost:8080`
- Content-Type：`application/json`

## 2. 接口列表

### 2.1 聊天接口
- URL：`POST /chat`
- Request
```json
{
  "user_id": "u001",
  "session_id": "default",
  "message": "劳动合同到期不续签有赔偿吗？"
}
```
- Response
```json
{
  "response": "律师回答内容...",
  "sources": [
    {
      "title": "劳动合同法第46条",
      "content": "有下列情形之一的...",
      "source": "法律条文",
      "score": 0.88
    }
  ]
}
```

### 2.2 查询会话历史
- URL：`GET /history/{user_id}/{session_id}`
- Response
```json
{
  "user_id": "u001",
  "session_id": "default",
  "messages": [
    {"role": "user", "content": "问题", "timestamp": 1711111111},
    {"role": "assistant", "content": "回答", "timestamp": 1711111112}
  ]
}
```

### 2.3 查询用户会话列表
- URL：`GET /sessions/{user_id}`
- Response
```json
{
  "user_id": "u001",
  "sessions": ["default", "case_001"]
}
```

### 2.4 创建会话
- URL：`POST /create_session`
- Request
```json
{
  "user_id": "u001",
  "session_id": "case_001"
}
```
- Response
```json
{
  "ok": true,
  "user_id": "u001",
  "session_id": "case_001"
}
```

### 2.5 清空会话
- URL：`POST /clear_session`
- Request
```json
{
  "user_id": "u001",
  "session_id": "case_001"
}
```
- Response
```json
{
  "ok": true,
  "user_id": "u001",
  "session_id": "case_001"
}
```

### 2.6 健康检查
- URL：`GET /health`
- Response
```json
{"status":"ok"}
```

## 3. 错误码约定
- 200：成功
- 400：参数错误（建议后续补充）
- 500：服务内部错误（当前实现）

## 4. Postman 测试建议
- 创建 Collection：`RAG_Lawyer_System`
- 配置变量：`baseUrl=http://localhost:8080`
- 覆盖场景：
  - 正常问答
  - 无命中问答
  - 多用户会话隔离
  - 清空会话后再次查询
